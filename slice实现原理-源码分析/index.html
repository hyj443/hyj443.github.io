为什么需要切片

几乎所有编程语言都有数组，go 为什么还要设计切片？数组有什么局限性？

数组的长度属于数组类型的一部分，只有数组内部元素类型和数组长度都一致时，两个数组才是同一类型。

一旦声明了一个数组，它的大小和内部元素的类型就不能改变，不能随意地向数组添加任意多个元素。

数组的大小是确定的，如果需要数组存储大量的数据，就要提前指定一片很大的内存。

go 语言中的函数传参是值传递，数组作为参数在函数中传递时，要对它进行拷贝，造成大量的内存浪费。

切片是对数组的抽象和封装，是变长的序列，序列中每个元素具有相同的类型，可以向切片中追加元素，它会在余量不足时自动扩容。

Data 是一片连续的内存空间，可以存储切片中的全部元素，数组中的元素只是逻辑上的概念，底层存储都是连续的，可以理解成切片是一片连续的内存空间加上长度和容量的标识

切片引入了一个抽象层，提供了对数组中部分连续片段的引用，我们可以在运行时修改它的长度和范围，当切片底层的数组长度不足时，会触发扩容，切片指向的数组可能会变化，不过在上层看来，切片没有发生变化，上层只需要和切片打交道，不关心底层数组的变化

编译器在编译期间简化了获取数组大小，读取数组中的元素等操作
因为数组的内存固定且连续，多数操作都会直接读写内存的特定位置
但是切片是运行时才会确定的结构，所有操作还需依赖 go 语言的运行时，下面会结合运行时介绍切片常见的操作背后的原理。

SliceMake 操作会接收 4 个参数而创建新的切片：元素类型 数组指针 切片大小 切片容量

这也是切片的几个字段，使用下标初始化切片不会拷贝原数组或原切片中的数据，而是创建一个指向原数组的切片结构体，所以修改新切片的数据也会修改原切片。

old 是原切片，new 是下标截取 old 的新切片，它们二者的地址正好相差 8 个字节，这不是偶然，是因为都指向了相同的数据源，刚好相差 int64 的大小

分析了主要由编译器处理的分支后，回到用于创建切片的运行时函数 makeslice，它主要是计算切片占用的内存空间，并申请在堆上的一片连续的内存

虽然编译期间可以检查出很多错误，但在创建切片的过程如果发生了以下错误，会直接触发运行时错误并 panic

  1. 内存空间的大小发生溢出
  2. 申请的内存大于最大可分配的内存
  3. 传入的长度小于 0，或长度大于容量

makeslice 最后调用 mallocgc 申请内存，这个函数实现比较复杂，如果遇到了比较小的对象会直接初始化在 go 语言的调度器中的 P 结构里，而大于 32KB 的对象会在堆上初始化（需要了解 Go 的内存分配器）

$ 内存空间 = 切片中元素的大小 × 切片容量 $

OSLICEHEADER 操作会创建 reflect.SliceHeader 结构体，其中包含数组指针，切片长度和容量，它是切片在运行时的表示：

不同的切片所对应的底层 Data，可能指向的是同一个数组，所以在切片赋值、重新进行切片操作时，使用的是同一个数组，没有复制原来的元素，从而减少内存的占用，提高效率。

但如果有一个切片修改内部的元素，其他切片也会受影响，所以在切片作为参数在函数间传递时要小心，尽可能不要修改原切片内的元素。

切片的本质是 SliceHeader 结构体，在函数间传递切片，传递的是 SliceHeader 的副本，而不是底层数组的副本，这时切片的优势就体现出来了，因为 SliceHeader 的副本内存占用很小，即使是一个非常大的切片（即底层数组很大），也最多只占用 24 个字节的内存。

SliceHeader 的三个字段的类型分别是 uintptr、int 和 int，在 64 位的机器上，最多也就是 int64 类型，一个 int64 占 8 个字节，所以一共占 24 个字节。

3.2.5 拷贝切片 #
copy (a, b)，编译期间的 cmd/compile/internal/gc.copyany 也会分两种情况进行处理拷贝操作，如果当前 copy 不是在运行时调用的，copy (a, b) 会被直接转换成下面的代码：

n := len(a)
if n > len(b) {
n = len(b)
}
if a.ptr != b.ptr {
memmove(a.ptr, b.ptr, n*sizeof(elem(a)))
}
Go
上述代码中的 runtime.memmove 会负责拷贝内存。而如果拷贝是在运行时发生的，例如：go copy (a, b)，编译器会使用 runtime.slicecopy 替换运行期间调用的 copy，该函数的实现很简单：

func slicecopy(to, fm slice, width uintptr) int {
if fm.len == 0 || to.len == 0 {
return 0
}
n := fm.len
if to.len < n {
n = to.len
}
if width == 0 {
return n
}

size := uintptr(n) * width
if size == 1 {*(*byte)(to.array) = *(*byte)(fm.array)

}
return n

}

3.2.6 小结 #
切片的很多功能都是由运行时实现的，无论是初始化切片，还是对切片进行追加或扩容都需要运行时的支持，需要注意的是在遇到大切片扩容或者复制时可能会发生大规模的内存拷贝，一定要减少类似操作避免影响程序的性能。

  • 切片是 go 中重要的数据结构，维护了底层的内存，长度和容量
  • 切片与数组的赋值拷贝有明显的区别，切片在赋值拷贝与下标截断时，引用了底层的数组
  • 如果想完全复制切片，使用 copy 函数，它的逻辑是新建一片内存，拷贝进去，在极端的情况下需要考虑其对性能的影响
  • 字面量初始化切片的方式，数组存储于静态区，make 初始化切片时，如果 make 一个大于 64KB 的切片，会发生逃逸，在堆中初始化，在运行时调用 makeslice 创建，小于 64KB 的切片在栈中初始化
  • 切片 append，当容量超出当前容量，需要进行扩容，策略是
    • 如果新申请的容量大于旧容量的两倍，那就是新申请的容量
    • 否则
      • 如果旧切片的长度小于 1024，则就是旧容量的两倍
      • 否则，就从旧容量还是增长 25%，直到最后容量大于等于新申请的容量
    • 如果最终容量计算值溢出，则最终容量就是新申请的容量
  • append 返回的切片地址不一定是原来的，也不一定是新的内存地址，要小心这个可能遇到的陷阱，一般会使用 a = append (a, T) 的方式保证安全
